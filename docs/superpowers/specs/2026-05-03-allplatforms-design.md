# AllPlatforms.io — Design Spec
**Date:** 2026-05-03
**Status:** Approved

---

## Overview

A static reference site at allplatforms.io that aggregates technical specifications, size limits, and requirements for the major social media platforms. Primary goals: high SEO ranking for spec-related queries, zero ongoing server cost, ad monetization once traffic reaches threshold.

**Audience:** Content creators, marketers, and developers — both casual and technical users.
**v1 Platforms:** Facebook, Instagram, YouTube, TikTok, LinkedIn, X (Twitter)

---

## Architecture

### Approach: JSON data + local build script (static site generation)

Spec data lives in JSON files. A Node.js build script reads each JSON file, injects data into shared HTML templates, and writes fully static HTML to a `public/` output directory. The `public/` folder is committed to git and served by Cloudflare Pages.

**Why this approach:**
- Pure static HTML = reliable Google indexing (no JS rendering issues)
- Data-driven = updating a spec means editing JSON and running `node build.js`, not touching HTML
- Future-proof = the JSON files can eventually be written by a crawler instead of by hand
- No framework, no build dependencies beyond Node.js

## Data Structure

Each platform is one JSON file in `/data/`. Schema:

```json
{
  "name": "Facebook",
  "slug": "facebook",
  "color": "#1877F2",
  "description": "Specs and technical requirements for all Facebook content formats.",
  "lastUpdated": "2026-05-01",
  "sources": [
    { "label": "Facebook Help Center", "url": "https://..." }
  ],
  "specs": [
    {
      "category": "images",
      "label": "Image Specs",
      "priority": 1,
      "items": [
        {
          "name": "Feed Photo",
          "dimensions": "1200 × 628 px",
          "aspectRatio": "1.91:1",
          "maxFileSize": "30 MB",
          "formats": ["JPG", "PNG"],
          "notes": "Recommended for link previews"
        }
      ]
    },
    {
      "category": "video",
      "label": "Video Specs",
      "priority": 2,
      "items": [...]
    },
    {
      "category": "text",
      "label": "Text & Character Limits",
      "priority": 3,
      "items": [...]
    },
    {
      "category": "profile",
      "label": "Profile & Cover Assets",
      "priority": 4,
      "items": [...]
    },
    {
      "category": "ads",
      "label": "Ad Specs",
      "priority": 5,
      "items": [...]
    }
  ]
}
```

**Key decisions:**
- `priority` controls section order on the page — image specs render first (highest search volume)
- `sources` stores official spec URLs for manual verification and future automation
- `lastUpdated` is displayed on every page as a trust signal and signals freshness to Google
- Item fields vary by category: image items use `dimensions`/`aspectRatio`/`formats`; video items use `duration`/`resolution`/`codec`; text items use `limit`/`unit`; all items share `name` and optional `notes`. The template renders whichever fields are present — absent fields are simply not shown.

## Homepage Layout

URL: `allplatforms.io/`

- Grid of platform cards (one per platform)
- Each card: platform icon, name, one-line description, 2–3 top spec values as teasers
- Client-side text filter: type "video" to highlight platforms with video coverage
- SEO title: "Social Media Specs & Size Guide 2026 | AllPlatforms.io"

## SEO Implementation

Every page includes:

| Element | Value |
|---|---|
| `<title>` | "Facebook Specs & Size Guide 2026 \| AllPlatforms.io" |
| `<meta description>` | Unique per platform, includes key spec values |
| `<h1>` | Platform name only |
| `<h2>` | Each spec category label |
| `<h3>` | Each individual spec item name |
| Canonical URL | Self-referencing canonical on every page |
| Open Graph tags | Title, description, URL per page |
| JSON-LD structured data | WebPage schema |
| `sitemap.xml` | Auto-generated, includes `<lastmod>` from JSON `lastUpdated` |
| `robots.txt` | Allow all, point to sitemap |
| Internal links | Sidebar on every platform page links to all other platforms |

The build script auto-generates `sitemap.xml` listing all platform URLs with their last-modified dates.

## Update Workflow (3 phases)

### Phase 1 — Manual (now)
Edit the relevant JSON file, run `node build.js`, commit and push. The `sources` array in each JSON points to the official spec pages to check.

### Phase 2 — Change detection (soon)
`scripts/check-sources.js` — a local Node.js script run weekly (or on demand). It fetches each URL in every `sources` array, hashes the response body, and compares against stored hashes. Outputs a report:

```
[CHANGED] Facebook — feed-image-specs — review needed
[OK]      Instagram — video-specs
```

No automatic updates — just tells you where to look. Stored hashes live in `scripts/.source-hashes.json` (gitignored).

### Phase 3 — Crawler/parser (future)
Extend the check script to extract values and propose JSON diffs for review before applying. Out of scope for v1.

## Styling

- Tailwind CSS via CDN (consistent with existing projects)
- Clean, neutral design that works for both technical and non-technical users
- Platform brand color used as an accent on each platform page (`color` field in JSON)
- Mobile-responsive layout (sidebar stacks below content on mobile)
- Ad slots adapt: leaderboard collapses to responsive unit on mobile

## Out of Scope for v1

- User accounts or saved posts
- AI-powered rewrites or suggestions
- Community spec reporting
- Newsletter
- Blog/articles section (placeholder route reserved: `/blog/`)
- Tier 2 platforms (Pinterest, Snapchat, Threads, Reddit, WhatsApp)
